  shared library

  Opzione 2: MAUI Hybrid (piГ№ "ufficiale", piГ№ overhead)

  Usa .NET MAUI con una custom view che hosta il rendering raylib. PiГ№
  boilerplate ma ti dГ  accesso al lifecycle Android nativo. Sconsigliato per il
  tuo caso вҖ” troppo overhead per un gioco.

  Opzione 3: Esporta come Web (WebAssembly) e wrappa in WebView

  Raylib supporta Emscripten/WASM. Potresti:
  1. Compilare il gioco per WASM con dotnet workload install wasm-tools
  2. Wrappare in un'app Android con WebView

  Pro: minime modifiche al codice. Contro: performance peggiore, interazione

  Come funziona su Android

  1. MainActivity crea una SurfaceView per il rendering OpenGL
  2. Quando la surface ГЁ pronta, avvia un thread di gioco separato
  3. Il thread ottiene ANativeWindow dalla Surface Android
  4. Chiama il bridge nativo che crea il contesto EGL + OpenGL ES 2.0
  5. Inizializza rlgl (il layer GL di raylib) вҶ’ tutte le funzioni di disegno
  raylib funzionano
  6. Il game loop in C# chiama Game.Init() e poi frame update/draw come su
  desktop
  7. Il touch viene mappato come mouse automaticamente

  Prossimi passi per far girare su Android

  1. Installa i prerequisiti

  # .NET Android workload
  dotnet workload install android

  # Android SDK + NDK (tramite Android Studio o sdkmanager)
  sdkmanager --install "ndk;25.2.9519653" "platforms;android-24"

  2. Compila le librerie native

  cd Native/Android
  # Su Windows:
  build.bat
  # Su Linux/Mac:
  ./build.sh
  Questo genera libraylib.so e libplants_bridge.so in
  Plants.Android/libs/arm64-v8a/.

  3. Compila e deploya l'app Android

  cd Plants.Android
  dotnet build -t:Install  # Compila e installa su device/emulatore connesso

  - Shader: i fragment shader .frag potrebbero necessitare piccole modifiche per
